LITE 모드 기획 개요

[목표]

본 개편은 복잡한 옵션 구조로 인한 사용자 혼란을 해소하기 위해, 핵심 기능 중심의 LITE(이지) 모드를 도입하는 것을 목표로 함

[LITE 모드 도입을 위한 검토 과정]

1) 옵션 전수 검토 방식

  • 기존 플랫폼의 모든 옵션값 목록화
  • 각 옵션에 대해 아래 항목 기준으로 검토
사용 빈도 중요도 UX 복잡도 CS 발생률 자동화 가능성

[참고] 옵션 전수 검토 작업 화면 (작업 중)

2) LITE 모드 분류 기준

분류설명
유지LITE 모드에서도 반드시 필요한 핵심 옵션
자동화사용자가 선택하지 않아도 시스템이 자동 설정
숨김기본 모드에서만 노출
제외현재 사용성 및 효과 대비 제거 대상

3) 기획 방향성

LITE 모드는 기능 축소가 아닌, 사용자의 선택을 시스템이 대신해주는 버전으로 정의
  • 복잡한 설정은 기본 모드에서만 제공
  • 언제든 버전 전환 가능

[LITE 모드 기획안]

1) LITE 모드 정의

  • 토글 버튼을 통해 기본 모드 ↔ LITE 모드 전환
  • 사용자 선택에 따라 언제든 전환 가능
  • 버전 전환 시, 데이터 구조는 동일하게 유지

2) 기본 모드 vs. LITE 모드

항목기본 모드LITE 모드
옵션 수다수 노출최소화
수업 도구 선순위 적용 자율 설정최초 설정값 유지
알림 선순위 적용 자율 설정최초 설정값 유지
평가/과제 설정 선순위 적용 출제할 때마다 직접 설정최초 설정값 유지
처방학습자동 출제사용자 선택
사용자 부담높음낮음

[기본 동작 정책 정의]

1) 최초 진입 사용자 기준

  • AIDT 첫 진입 시 사용 모드 선택 가이드 팝업 노출
  • 기본 선택값은 LITE 모드
  • LITE 모드 유지 ↔ 기본 모드 전환 선택 가능

2) 재진입 사용자 기준

  • 팝업 재노출 X
  • 마지막 설정 모드 유지 적용
  • 우측 상단 [설정] 버튼 통해 변경 가능

3) LITE 모드 적용 이전 사용자 기준

  • 기본값: 기본 모드 유지
  • 최초 1회 LITE 안내 배너 또는 가이드 제공 가능
